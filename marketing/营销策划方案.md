# FreeJournal 营销策划方案

> 核心定位：独立开发者的免费工具，用真诚换真实反馈

---

## 一、目标与策略

### 当前阶段目标
- **首月目标**：500+ 下载，50+ 真实反馈
- **三月目标**：2000+ 下载，建立 200 人种子用户群
- **核心指标**：用户留存率 > 反馈数量 > 下载量

### 核心策略：免费 + 真诚 + 反馈驱动
不卖产品，卖故事。你不是一个公司在推广，你是一个开发者在分享自己做的工具。

## 三、渠道与内容策划

### 第一梯队：高转化率渠道（立即启动）

#### 1. V2EX — 程序员社区
**发帖标题**（选一个）：
- `独立开发两周，做了一个 macOS 语音输入法，完全免费，求反馈`
- `不想打字了，所以自己写了个语音输入法（macOS，免费开源）`
- `一个人做了两个 App，免费送给大家用，求骂`

**帖子结构**：
```
## 为什么做这个

我是一个独立开发者，平时写代码写文档打字太多，手腕疼。
试了苹果自带的听写、讯飞、Whisper，都有各种不满意的地方。
于是自己写了一个：VoiceJar。

## 它能做什么

- 按住 Fn 说话，松开就输入到光标位置，任何 App 都能用
- 实时识别，边说边出字
- AI 自动润色：修错字、加标点、去"嗯啊那个"
- 支持 DeepSeek、Claude、Gemini、Ollama（本地）
- 划词翻译，一个快捷键搞定

## 完全免费

没有试用期，没有内购，没有广告。
AI 用你自己的 API Key，花多少完全透明。

## 为什么免费

说实话，我是新手开发者，现在最需要的不是钱，是反馈。
哪里不好用、哪里让你想骂人，直接告诉我。

下载：https://freejournal.app/voicejar/
另一个日志 App：https://freejournal.app/babyjournal/

求反馈，求 star，求骂。
```

#### 2. 少数派 — 效率工具社区
**文章标题**：
- `按住说话，松开输入：一个独立开发者做的 macOS 语音输入法`
- `我不想打字了，所以自己做了个语音输入法（免费）`

**文章风格**：少数派用户喜欢有深度的体验文，建议写 1500-2000 字，包含：

## 五、传播策略：让用户帮你传播

### 策略 1：反馈即回报
- 在 App 设置页加一个"反馈"按钮，跳转到问卷/邮件
- 每条反馈必回复，让用户感到被重视
- 好的反馈截图发到社交媒体，@用户表示感谢

### 策略 2：参与感
- 让用户投票决定下一个功能（即刻/GitHub Issues）
- 在更新日志里 @提建议的用户："感谢 @xxx 的建议"
- 建一个 changelog 页面，每次更新都让用户看到进步

### 策略 3：口碑传播话术
在 App 里或网站上加一句：
> 如果你觉得好用，告诉一个朋友就是最好的支持。

### 策略 4：内容复用
一次开发故事 → 多平台分发：
- V2EX：技术向，讲实现
- 少数派：体验向，讲使用
- 即刻：情感向，讲动机
- 小红书：颜值向，讲设计
- 掘金：代码向，讲架构

## 九、风险与应对

| 风险 | 应对 |
|------|------|
| 下载量低 | 换渠道/换文案角度，测试哪个点最打人 |
| 负面反馈多 | 好事！说明有人在用。快速修复，公开回应 |
| 被抄袭 | 速度是护城河，持续迭代。考虑开源反而能建立社区 |
| 苹果审核被拒 | BabyJournal 走 App Store，VoiceJar 走官网直下载 |

## 执行建议

**推荐方案：逐步实施 + 阶段评估**

1. **Week 1**：V2EX + 即刻，验证产品吸引力
2. **评估**：看下载量和反馈质量，调整话术
3. **Week 2-3**：少数派 + 小红书 + Twitter，扩大覆盖
4. **评估**：哪个渠道转化率最高，集中资源
5. **Week 4**：建社群 + 提交收录网站，沉淀长尾流量

不要一次铺开所有渠道。先小范围测试，拿到数据再决定加注。
